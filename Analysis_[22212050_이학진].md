## 1. Introduction

본 문서는 **'왓 투 웨어(WhatToWear)'** 시스템의 개념화(Conceptualization) 단계 요구사항을 구체화하기 위해 작성된 분석(Analysis) 단계의 산출물이다. 향후 시스템 설계 및 구현의 명확한 기준선을 제공하기 위해, 본 시스템을 구성하는 핵심 데이터 객체의 정적 구조를 정의하는 도메인 모델(Domain models), 사용자와 시스템 간의 동적 상호작용 흐름을 기술하는 유스케이스 명세서(Use case specifications), 그리고 주요 기능의 시간적 메시지 전달 과정을 시각화한 시스템 시퀀스 다이어그램(System sequence diagram)을 명세한다.

<br>

### 1.1 System Summary (시스템 요약)
'왓 투 웨어'는 실시간 기상 데이터에 사용자 고유의 체질(열 민감도)과 외출 목적(TPO)을 결합하여 개인화된 옷차림을 추천하는 지능형 맞춤형 서비스이다. 기존 일기예보가 해결하지 못하는 주관적인 체감 온도의 차이를 데이터화하여 일상 속 옷차림에 대한 고민과 불편함을 근본적으로 해소하는 것을 목표로 한다.

<br>

### 1.2 Prominent Features (프로젝트 주요 특징)

### ✅ 유용성 (Usefulness)
객관적인 기온 수치만 제공하는 기존 일기예보의 한계를 극복하고, 사용자의 주관적인 체감 온도를 계산하여 상황에 맞는 최적의 옷차림을 즉각적으로 제시함으로써 매일 아침의 고민과 외출 시의 불편함을 해소한다.

### ✅ 개인화 (Personalization)
외출 후 사용자가 남긴 체감 온도 피드백을 시스템이 정량적 데이터로 수집하고, 추천 알고리즘의 가중치를 스스로 업데이트(자가 학습)하여 사용할수록 개인에게 완벽히 최적화되는 경험을 제공한다.

### ✅ 확장성 (Expandability)
개인 옷장 데이터를 활용하는 것을 넘어, 향후 외부 의류 커머스 API와 연동한 의류 추천 및 판매로 확장할 수 있다. 또한 미세먼지나 강수량 등 복합 기상 데이터를 추가 연동하여 알고리즘을 고도화할 수 있는 유연한 구조를 가진다.

* **기능:** 사용자가 당일 추천받은 옷차림에 대해 실제 느낀 온도감을 평가하여, 개인 맞춤형 추천 알고리즘을 고도화한다.
* **상세 설명:** 메인 화면에서 피드백 버튼을 누르면 팝업 형태로 나타나며, 사용자는 5단계의 직관적인 선택지로 그 날의 옷차림에 대한 평가를 남길 수 있다. 제출된 피드백 데이터는 백엔드 서버로 전송되어 사용자의 '개인 체질 가중치'를 재계산하는데 쓰이며, 이를 통해 다음 번 날씨 기반 추천 시 더욱 정교하고 완벽하게 개인화된 옷차림을 제안하는 핵심 동력으로 활용된다.

<br>

### 4.3. Responsive Layout Strategy
* **Mobile View:** 한 손 조작이 가능하도록 하단 네비게이션 바와 수직형 리스트 구조를 채택하였다.
* **Desktop View:** 화면 중앙에 센터 카드 레이아웃을 적용하여 모니터 환경에서도 시각적 안정감을 확보하였다. 데스크톱 환경에서는 좌우 여백을 활용하여 콘텐츠 집중도를 높이는 전략을 사용한다.

## 5. Glossary

| 용어(Term) | 정의 및 프로젝트 내 역할 |
| :--- | :--- |
| **API(Application Programming Interface)** | 서로 다른 소프트웨어 시스템 간에 데이터를 주고 받기 위한 규격. 본 프로젝트에서는 기상 데이터 및 주소 정보를 외부 서버로부터 호출하는데 사용됨. |
| **NX / NY(기상청 격자 좌표)** | 대한민국 기상청에서 단기예보 조회를 위해 사용하는 고유의 평면 좌표 체계. 위경도 좌표를 일정한 공식을 통해 변환하여 사용함. |
| **다음(Daum) 우편번호 서비스** | 사용자가 검색을 통해 정제된 문자열 주소(법정동)를 쉽게 획득할 수 있도록 카카오에서 제공하는 프론트엔드 전용 주소 검색 UI 라이브러리. |
| **카카오 로컬(Local) API** | 우편번호 서비스로 획득한 텍스트 주소를, 기상청 API 연동에 필수적인 지리적 좌표(위도, 경도)로 변환해 주는 서버 간(REST) API. |
| **반응형 웹(Responsive Web)** | 접속하는 기기(모바일, PC 등)의 화면 해상도에 따라 레이아웃이 유동적으로 변하는 웹 설계 방식. |
| **가중치(Weight)** | 사용자의 체질 피드백을 수치화한 값. 기온 데이터에 이 값을 연산하여 개인별 '보정 기온'을 산출함. |
